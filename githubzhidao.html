🚀 将HTML文件部署到GitHub Pages指南 📄

一步步教您如何轻松地将您的静态HTML页面发布到互联网上！

1

准备工作

在开始之前，请确保您已准备好以下两项：

  • 一个 GitHub 账户。如果您还没有，可以访问 GitHub官网 免费注册一个。
  • 您想要部署的 HTML 文件。例如，一个名为 `index.html` 的文件。确保您的 HTML 文件是完整的，并且可以在本地浏览器中正常显示。
2

创建新的 GitHub 仓库 (Repository)

代码仓库是您存放项目文件的地方。

  1. 登录您的 GitHub 账户。
  2. 点击页面右上角的 "+" 图标，然后选择 "New repository"。
  3. 为您的仓库命名。一个常见的做法是使用 `.github.io` 作为仓库名，这样您的主页链接会更简洁。 如果您只是想部署单个项目，任何名称都可以，例如 `my-awesome-page`。
  4. （可选）添加描述。
  5. 选择 "Public" (公开仓库)，因为 GitHub Pages 免费版只支持公开仓库。
  6. （可选）勾选 "Add a README file" (这会创建一个说明文件，是个好习惯)。
  7. 点击 "Create repository" 按钮。
3

上传 HTML 文件

将您的 HTML 文件添加到新创建的仓库中。

  1. 在您的新仓库页面，点击 "Add file" 按钮，然后选择 "Upload files"。
  2. 您可以将 HTML 文件拖拽到指定区域，或者点击 "choose your files" 选择文件。

    💡 提示: 如果您的 HTML 文件名为 `index.html`，它将成为您网站的默认主页。

  3. 文件上传后，在页面下方找到 "Commit changes" (提交更改) 部分。
  4. （可选）填写提交信息，简要描述您所做的更改 (例如："Add initial HTML file")。
  5. 点击 "Commit changes" 按钮。
4

启用 GitHub Pages

这是将您的仓库变成一个可访问网站的关键步骤。

  1. 在您的仓库页面，点击顶部的 "Settings" (设置) 标签页。
  2. 在左侧导航栏中，找到并点击 "Pages" (页面) 选项。
  3. 在 "Build and deployment" (构建和部署) 部分下的 "Source" (源) 设置中，选择您要部署的分支。通常是 `main` (或 `master`，取决于您的默认分支名称)。
  4. 选择文件夹。通常是 `/ (root)`，表示使用仓库的根目录。
  5. 点击 "Save" (保存) 按钮。
  6. 页面刷新后，您应该会看到一条消息，说明您的站点已准备好发布，或者正在构建中。

    ⏳ 请注意: GitHub Pages 可能需要几分钟时间来构建和部署您的站点。请耐心等待。

5

访问您的在线 HTML 页面 🎉

一旦您的站点成功发布，GitHub Pages 设置页面会显示您的站点链接。

  • 链接的格式通常是： https://<your-username>.github.io/<repository-name>/
  • 如果您将仓库命名为 `.github.io` 并且 HTML 文件是 `index.html`，那么链接将是： https://<your-username>.github.io/
  • 点击该链接，您就可以在浏览器中看到您部署的 HTML 页面了！
前往 GitHub 开始实践
💡

额外提示

  • 自定义域名: GitHub Pages 支持绑定您自己的域名，让您的网站更专业。您可以在 "Pages" 设置中找到相关选项。
  • 更新内容: 如果您想更新您的 HTML 页面，只需将修改后的文件重新上传到 GitHub 仓库并提交更改即可。GitHub Pages 会自动重新部署。
  • CSS 和 JavaScript: 如果您的 HTML 文件引用了外部的 CSS 或 JavaScript 文件，请确保这些文件也一起上传到仓库中，并使用相对路径正确引用。例如，如果您的 CSS 文件在 `styles` 文件夹下，路径应为 `styles/style.css`。
  • 故障排除: 如果您的页面没有按预期显示，请检查：
    • HTML 文件名是否为 `index.html` (如果希望它是默认页面)。
    • GitHub Pages 设置中的分支和文件夹是否正确。
    • 浏览器缓存，尝试强制刷新 (Ctrl+Shift+R 或 Cmd+Shift+R)。
    • 浏览器开发者工具 (F12) 中的控制台是否有错误信息。

祝您部署顺利！通过 GitHub Pages 分享您的创意吧！
